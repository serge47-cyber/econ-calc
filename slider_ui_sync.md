Цей звіт підготовлено за результатами статичного та динамічного аудиту інтерфейсу користувача та розрахункової логіки інтерактивного посібника [economic-tasks.html](file:///Users/serhiikhudolii/econ-calc/economic-tasks.html).

## 1. Аналіз HTML/CSS на наявність «мертвого» коду

Ми провели сканування всіх елементів вводу (`<input>` та `<select>`) в HTML-розмітці та зіставили їх із викликами `document.getElementById()` у JavaScript-коді.

### Результати аналізу:
1. **Відсутність неактивних HTML-елементів:** Всі знайдені числові поля вводу (`type="number"`), повзунки (`type="range"`) та списки вибору (`<select>`) мають відповідні JS-прив'язки. Кожен елемент бере участь у розрахунках принаймні однієї фінансово-економічної функції.
2. **Відсутність зламаних JS-посилань:** Усі виклики `document.getElementById()` в JavaScript звертаються до реально існуючих в HTML-коді елементів. Немає ситуацій, коли скрипт намагається зчитати дані з раніше видаленого текстового поля.
3. **Співіснування повзунків та текстових віконець:** 
   - Модулі **«Складні відсотки»**, **«Ануїтет»**, **«Амортизація»**, **«Беззбитковість»**, **«Direct vs Absorption»** та **«Бюджет і відхилення»** працюють **виключно на повзунках** (`type="range"`). Вони не містять дублюючих чи прихованих текстових полів для тих самих параметрів.
   - Модулі **«Рентабельність»**, **«Ліквідність»**, **«WACC»**, **«Облігації»**, **«Акції»**, **«Сценарний аналіз»**, **«Горизонтальний аналіз»** та **«Стійкість»** працюють **виключно на числових полях вводу** (`type="number"`). Для них повзунки не створювалися, оскільки студентам необхідно вводити точні, некруглі значення фінансової звітності підприємств.
   - Модулі **«NPV / IRR»** та **«Порівняння проектів»** використовують гібридний підхід: ставка дисконтування задається повзунком (`n_r`/`pc_r`), а початкові інвестиції та грошові потоки по роках (`n_cf1`–`n_cf6`) вводяться вручную через числові поля для точності.

### Рекомендація щодо оптимізації верстки:
Оскільки мертвих елементів не виявлено, видалення будь-якого `<input>` призведе до порушення роботи відповідної функції. Проте для покращення UX (досвіду користувача) рекомендується реалізувати **двосторонню синхронізацію** (Slider + Number input) для ключових параметрів. Це дозволить студентам як перетягувати повзунок, так і вводити точні значення вручну.

## 2. Перевірка розрахункових функцій на зчитування з діючих повзунків

Ми детально перевірили, з яких саме елементів зчитують дані ключові функції:

1. **`ci()` (Складні відсотки):**
   - Зчитує з повзунка `c_pv` (Початкова сума) -> `type="range"` (рядок 430)
   - Зчитує з повзунка `c_r` (Ставка) -> `type="range"` (рядок 431)
   - Зчитує з повзунка `c_n` (Роки) -> `type="range"` (рядок 432)
   - Зчитує з випадаючого списку `c_m` -> `select` (рядок 433)
   - **Статус:** Повна відповідність.

2. **`an()` (Ануїтет / Кредит):**
   - Зчитує з повзунка `a_pv` (Сума кредиту) -> `type="range"` (рядок 523)
   - Зчитує з повзунка `a_r` (Ставка) -> `type="range"` (рядок 524)
   - Зчитує з повзунка `a_n` (Роки) -> `type="range"` (рядок 525)
   - Зчитує з повзунка `a_dp` (Перший внесок %) -> `type="range"` (рядок 526)
   - **Статус:** Повна відповідність.

3. **`dep()` (Амортизація):**
   - Зчитує з повзунка `d_cv` (Первісна вартість) -> `type="range"` (рядок 719)
   - Зчитує з повзунка `d_sv` (Ліквідаційна вартість) -> `type="range"` (рядок 720)
   - Зчитує з повзунка `d_n` (Роки) -> `type="range"` (рядок 721)
   - Зчитує з повзунка `d_k` (Коефіцієнт прискорення) -> `type="range"` (рядок 722)
   - **Статус:** Повна відповідність.

4. **`be()` (Точка беззбитковості):**
   - Зчитує з повзунка `b_fc` (Постійні витрати) -> `type="range"` (рядок 811)
   - Зчитує з повзунка `b_p` (Ціна одиниці) -> `type="range"` (рядок 812)
   - Зчитує з повзунка `b_vc` (Змінні витрати) -> `type="range"` (рядок 813)
   - Зчитує з повзунка `b_q` (Фактичний обсяг) -> `type="range"` (рядок 814)
   - **Статус:** Повна відповідність.

5. **`calcNPV()` (NPV / IRR):**
   - Зчитує з числового поля `n_ic` (Інвестиція) -> `type="number"` (рядок 616)
   - Зчитує з повзунка `n_r` (Ставка дисконтування) -> `type="range"` (рядок 617)
   - Зчитує з числових полів `n_cf1`–`n_cf6` (Потоки) -> `type="number"` (рядки 618-623)
   - **Статус:** Повна відповідність.

function syncInput(sliderId, inputId) {
  document.getElementById(inputId).value = document.getElementById(sliderId).value;

function syncSlider(inputId, sliderId) {
  let val = parseFloat(document.getElementById(inputId).value);
  const slider = document.getElementById(sliderId);
  if (isNaN(val)) return;
  // Обмеження значень відповідно до лімітів повзунка
  val = Math.max(parseFloat(slider.min), Math.min(parseFloat(slider.max), val));
  slider.value = val;
}
